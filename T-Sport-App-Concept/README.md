# Концепция мобильного приложения «Т-Спорт»

Проект по расширению экосистемы **Т-Банка** через стратегию **Lifestyle Banking**. Основная цель — интеграция финансовых сервисов в повседневную активность клиента для повышения лояльности и сбора более глубоких данных о пользователе.

---

## 📌 Проблематика и контекст

Банк сталкивается с «слепой зоной» вне транзакций: данные о здоровье и активности клиентов оседают в сторонних сервисах (Apple Health, Strava). 
**Т-Спорт** решает задачу перехода от чисто финансовых сервисов к управлению качеством жизни (Health & Wealth), что увеличивает LTV клиента на 20-30%.

---

## 🛠 Аналитическая работа в проекте

В рамках кейса были проработаны следующие аспекты:

### 1. Проектирование процессов (BPMN)
Разработана логика **автоматического начисления вознаграждений** за физическую активность. Схема включает:
* Интеграцию с внешними API (Apple/Google Health).
* Проверку выполнения условий «Цели недели».
* Триггеры для начисления кэшбэка и пуш-уведомлений.
*(Файл процесса: `кейс BA.drawio`)*

### 2. Состав MVP и функциональные блоки
* **T-ID:** Бесшовная авторизация в контуре банка.
* **Activity Tracker:** Сбор данных о шагах и калориях.
* **Gamification:** Механика челленджей для удержания (Retention).
* **Content Engine:** База знаний по ЗОЖ для привлечения трафика (снижение CAC).

### 3. Бизнес-анализ и риски
* **Синергия:** Описаны механики кросс-продаж для Т-Страхования (скидки на ДМС) и подписки Т-Pro.
* **Риски:** Проведен анализ технической зависимости от вендоров ОС и юридических ограничений (ФЗ-152 «О персональных данных»).

---

## 📊 Ключевые метрики (Прогноз)
* **LTV:** Рост на 25% в долгосрочной перспективе.
* **Retention:** Повышение частоты использования приложения (Daily Active Users).
* **Churn Rate:** Снижение оттока за счет глубокого вовлечения в экосистему.
